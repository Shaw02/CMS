	SSE2(SIMD)命令により、高速にAES暗号・復号を行うソフトウェアです。
	・Ｃ言語（Visual C++）での、暗号・復号ソフトの開発例の参考
	・単一ファイルに、ＡＥＳによる暗号・復号を行いたい。
	等の用途にご使用下さい。
	※尚、SIMD化されているのは、ＡＥＳ暗号・復号の処理ルーチンのみです。

	●特徴
	・SSE2(SIMD)命令を用いた、高速な暗号・復号処理。
	・暗号文、暗号鍵の各ファイルはRFC勧告に従った標準形式で出力します。
	　※拡張子は標準化されていない為、独自拡張子です。
	　　【暗号文ファイル】
		PKCS#7 Cryptographic Message Syntax (CMS)
		※暗号化データ(Encrypted-data)タイプを出力します。
	　　【暗号鍵ファイル】
		PKCS#8 Public-Key Cryptography Standards
		Private-Key Information Syntax Specification Version 1.2
		※対称鍵における秘密鍵用の規格ですが、共通鍵を格納しています。
	・暗号の安全性を高めています。
	　　(1) 初期化ベクタ(IV)は、毎回、擬似乱数により生成し、その為、
		同じ暗号鍵でも、毎回、異なる暗号文が生成されます。
	　　(2) ワンタイム暗号鍵及びIVは、擬似乱数"MT"の出力を、
		ハッシュ"SHA-256"化した値を使用し、推定し難い値となります。

	●本ソフトを構成するクラスについて
	本ソフトウェアは、以下の機能（クラス）が実装されています。
	　・AES		･･･ 暗号関数（AES-128, AES-192, AES-256）
	　・SHA-256	･･･ ハッシュ関数（SHA-256）
	　・MT		･･･ 擬似乱数関数（Mersenne Twister）
	アプリケーションというより、技術資料的なものとしての意味の方が強いです。
	使い勝手とかは、全然考えていません。ＣＵＩですし。
	（というか。VC++.net 2008 Expressなので、ＧＵＩは作れない。）

	●注意
	本ソフトウェアでは、暗号鍵の管理は行いません。
	暗号鍵の管理は、ご使用者にて、適切な方法にて行ってください。
	例えば、第三者に暗号鍵を渡す際は、以下に示す例の様に、
	第三者が本AES暗号鍵を入手できない方法で渡してください。
	　例1 ) 受信相手のみが知りえる『キーフレーズ』を暗号鍵に設定する。
	　例2 ) 公開鍵暗号方式にて本AES暗号鍵を暗号化し、電気通信(*)にて渡す。
	　例3 ) 破棄可能な磁気メディア等にて本AES暗号鍵を渡す。
	（* e-mailやFTPは平文で送受信される為、盗聴された場合、漏洩します。）
